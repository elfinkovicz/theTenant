VIRALTENANT INFRASTRUCTURE - MODULE ÜBERSICHT
==============================================

CORE MODULES:
-------------
central-auth          → Cognito User Pool + API Gateway REST API für Authentifizierung
lambda-authorizer     → Custom Authorizer für API Gateway, prüft JWT-Token und Tenant-Zugehörigkeit
lambda-layers         → Shared Node.js Dependencies für alle Lambda-Funktionen

TENANT MANAGEMENT:
------------------
tenant-management     → DynamoDB-Tabellen für Tenants, User-Tenants und Assets
tenant-registration   → E-Mail-Verifizierung und Tenant-Erstellung bei Registrierung
tenant-api            → CRUD-Operationen für Tenants (GET, POST, Subdomain-Check)

PAGE TAB MODULES (Frontend-Seiten):
-----------------------------------
tenant-frontend       → /Home: Hero-Content, Themes, Design-Settings und Werbebanner
tenant-live           → /Live: Live-Streaming Einstellungen, Thumbnails, Overlays
tenant-videos         → /Videos: Video-Content, Thumbnails, Kategorien
tenant-shop           → /Shop: Produkte, Produktbilder, Kategorien
tenant-events         → /Events: Events, Event-Bilder, Banner
tenant-newsfeed       → /Newsfeed: Posts, Post-Bilder, Medien
tenant-channels       → /Channels: Kanäle, Icons, Banner
tenant-team           → /Team: Team-Mitglieder, Avatare
tenant-contact        → /Contact: Kontaktinfos, Hintergrundbilder

BILLING:
--------
billing-api           → Stripe-Integration für Zahlungen und Rechnungen

STORAGE & CDN:
--------------
s3-tenant-storage     → S3-Buckets für Creator-Assets und temporäre Uploads
s3-bucket-policies    → CloudFront-Zugriffsrichtlinien für S3-Buckets
cloudfront            → CDN-Distribution mit Custom Domains und SSL

DNS & ROUTING:
--------------
route53               → DNS-Zone und SSL-Zertifikat für viraltenant.com + Wildcard

s3://viraltenant-creator-assets-production/

        ├── channels/       → Kanal-Icons, Banner (tenant-channels)
        ├── team/           → Team-Avatare (tenant-team)

DATENFLUSS:
==============================================
1. Admin lädt Asset hoch:
   Frontend → POST /upload-url → Lambda → S3 Presigned URL
   Frontend → PUT direkt zu S3 mit Presigned URL
   Frontend → PUT /[tab] mit S3 Key → Lambda → DynamoDB

2. User lädt Seite:
   Frontend → GET /[tab] → Lambda → DynamoDB
   Lambda konvertiert S3 Keys zu CloudFront URLs
   Frontend zeigt Bilder via CloudFront CDN
